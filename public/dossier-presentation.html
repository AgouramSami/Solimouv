Architecture
Justification des choix techniques
Chaque décision est motivée par les contraintes du projet : MVP rapide, performance mobile, pérennité, accessibilité.
🧡

Svelte + Vite — Frontend

Framework compilé ultra-léger, idéal pour PWA mobile-first

Front
  • Pas de Virtual DOM → rendu natif plus rapide
  • Bundle < 50 KB gzip pour le socle
  • Vite : HMR instantané, build optimisé
  • PWA plugin officiel (@vite-pwa) natif
  • Syntax simple = onboarding équipe rapide
  • Score Lighthouse > 90 out-of-the-box
🦀

Rust — Backend API

Performance native, sécurité mémoire, compilable en WASM

Back · Bonus
  • Axum : framework async HTTP haute perf
  • Sécurité mémoire sans GC (0 data race)
  • Moteur de matching sportif embarqué
  • Compilation WASM → logique côté client possible
  • Temps de réponse API < 5ms median
  • Défi référent : points bonus significatifs
📱

PWA — Progressive Web App

Installable sans store, accessible à tous les publics

Mobile-first
  • Installation directe → pas de barrière App Store
  • Fonctionne offline (service worker + cache)
  • Une seule base de code web + mobile
  • Notifications push natives
  • Public inclusif : pas de smartphone récent requis
  • SEO indexable (contrairement à une app native)
☁️

Render — Hébergement

PaaS adapté aux MVPs associatifs, déploiement en 1 clic

Hosting MVP
  • Free tier suffisant pour le MVP (trafic faible)
  • Déploiement Rust natif sans Docker requis
  • Auto-deploy sur push GitHub
  • SSL automatique, domaine custom inclus
  • Scale horizontal si l'édition grossit
  • Coût maîtrisé : 0€ → 7$/mois selon usage

💡 Économie structurelle
La PWA évite les coûts de développement App Store (iOS ~99$/an, Android ~25$) et les contraintes de review. L'architecture statique (Svelte buildé) réduit les coûts serveur de 80% vs une app full-server.
